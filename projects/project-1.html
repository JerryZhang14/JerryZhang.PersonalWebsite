Project 01

FPGA VGA Tron-Style Game

Code on github

This is a Tron-inspired game built on an FPGA and displayed on a VGA monitor. Players control a moving line inside a grid arena. As you move, you leave a trail behind you. The goal is to avoid crashing into walls or any trailâ€”once you collide, the round ends.

Instead of running on a computer program, this game runs directly in digital hardware. The FPGA generates the VGA video signal and updates the game in real time, frame by frame, without relying on a CPU or graphics libraries.

FPGA HDL VGA FSM Real-Time Graphics
Hardware Driven No CPU or graphics libraries used; all timing and logic are synchronous hardware blocks.
Persistence Logic Custom memory mapping tracks the history of player movements to generate the lethal "light trails."

Build Breakdown

01

VGA Timing Pipeline

  • Custom H/V sync timing using pixel counters to generate a stable 60Hz signal.
  • Synchronization of active video regions with RGB output.
02

State Persistence

  • Grid-based arena occupancy stored in memory to track player trails.
  • Real-time rendering of trails based on current memory state.
03

Collision Engine

  • Integrated hardware logic checking if the player's next pixel is already "occupied."
  • Immediate FSM state transition to Game Over upon collision detection.

Skills Demonstrated

  • HDL-based FPGA system design (Verilog/VHDL)
  • Timing-accurate VGA signal generation
  • FSM-controlled real-time game logic
  • Synchronous hardware memory management

Challenges & Solutions

  • Stable Video: Meticulously validated sync pulse boundaries to ensure flicker-free output.
  • Hardware Debouncing: Implemented signal stabilization for physical button inputs to prevent false moves.

Final Outcome

A fully playable Tron-style game running entirely in hardware, demonstrating end-to-end FPGA system design and mastery of real-time graphics control without traditional software abstraction.
